#### Paso 3: Subir tu código a GitHub

**Opción A - Con GitHub Desktop (MÁS FÁCIL):**
```
1. Descarga GitHub Desktop: https://desktop.github.com
2. Instala y abre GitHub Desktop
3. Clic en "File" → "Add Local Repository"
4. Selecciona la carpeta donde descomprimiste tu app
5. Si dice "not a git repository", clic en "Create Repository"
6. Clic en "Publish repository"
7. Selecciona tu cuenta y el nombre del repo
8. Clic en "Publish Repository"
```

**Opción B - Con Terminal (PARA DESARROLLADORES):**
```bash

# Agregar todos los archivos
git add .

# Hacer commit
git commit -m "Primera versión de mi app"

# Conectar con GitHub (reemplaza TU_USUARIO y TU_REPO)
```

## 🔧 **SOLUCIÓN DE PROBLEMAS COMUNES**

### ❌ "No veo el banner de instalación en mi celular"

**Solución:**
```
1. Asegúrate de que la URL empiece con HTTPS (Vercel lo hace automáticamente)
2. Android: Usa Chrome (no Firefox, Opera, etc.)
3. iPhone: Usa Safari (no Chrome, Firefox, etc.)
4. Recarga la página (desliza hacia abajo)
5. Usa el método manual:
   - Chrome: ⋮ → "Agregar a pantalla de inicio"
   - Safari: Compartir → "Agregar a la pantalla de inicio"
```

---

### ❌ "La app no cambia de Paciente a Cliente"

**Solución:**
```
1. El nombre debe contener palabras médicas para decir "Paciente":
   - consultorio, dental, clínica, doctor, médico, salud, hospital
2. Si NO tiene esas palabras, dirá "Cliente"
3. Después de cambiar el nombre:
   - Recarga la página (Ctrl+R o Cmd+R)
   - O cierra y vuelve a abrir la app
```

---

### ❌ "Los datos se borran cuando cierro la app"

**Solución:**
```
Esto es NORMAL si no has conectado Supabase.
Los datos se guardan en localStorage del navegador.

Para NO perder datos:
- No borres el caché del navegador
- No uses modo incógnito
- Los datos son específicos de cada dispositivo

Para datos en la nube (opcional):
- Lee el archivo: INSTRUCCIONES-FINALES.md
- Sigue los pasos para conectar Supabase
```

---

### ❌ "Error al desplegar en Vercel"

**Solución:**
```
1. Verifica que el archivo vercel.json existe en tu proyecto
2. Verifica que package.json existe
3. En la configuración de Vercel:
   - Build Command: pnpm build
   - Output Directory: dist
   - Install Command: pnpm install
4. Si el error persiste, usa Node 18.x en Vercel:
   - Settings → General → Node.js Version → 18.x
```
